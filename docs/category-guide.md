# Category guide

Categories describe **why** someone would use something, not **what** Open WebUI primitive it is.

| Category | When to use |
|----------|-------------|
| **Coding** | Helpers for reading, editing, building, reviewing, or running code. |
| **Research** | Web search, summarization, citations, comparing sources. |
| **RAG** | Chunking, embeddings, vector stores, retrieval quality, document chat. |
| **Automation** | Triggering workflows, APIs, schedulers, glue between systems. |
| **Productivity** | Writing, notes, calendars, personal knowledge—not only coding. |
| **Multi-agent** | Routing, delegation, multiple models or roles in one workflow. |
| **Self-hosting** | Running Open WebUI and dependencies on your own infra. |
| **Security** | Auth, secrets handling, sandboxing, supply-chain awareness. |

An entry can appear in multiple categories. Prefer **two or fewer** primary categories in the entry header to keep browsing useful.

## Relationship to stacks

- **Categories** = broad intent.
- **Stacks** = a named recipe (goal, components, trade-offs). See [../stacks/](../stacks/).

When a stack doc exists, link it from **Stack fit** in the catalog entry.
